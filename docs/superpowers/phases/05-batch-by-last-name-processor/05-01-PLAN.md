<objective>
Process all 55.5M name index records through Splink predict+cluster in memory-safe batches grouped by last_name, producing `lake.federal.resolved_entities` with cluster_id and match_probability for every record.

Purpose: The full 55.5M records OOM if processed at once. Batching by last_name is safe because BOTH blocking rules require last_name match — records with different last names are never compared. Pack multiple last names per batch (up to ~500K records) for efficiency.
Output: `lake.federal.resolved_entities` table in DuckLake with cluster_id, unique_id, source_table, first_name, last_name, address, city, zip, match_probability.
</objective>

  <action>
  Create `scripts/batch_resolve_entities.py` that processes all 55.5M records in batches:

  1. Connect to DuckLake via `_connect_ducklake()` (same as train script).

  2. Get last_name frequency distribution:
     ```sql
     SELECT last_name, COUNT(*) as cnt
     FROM lake.federal.name_index
     GROUP BY last_name
     ORDER BY last_name
     ```

  3. Pack last names into batches using greedy bin-packing:
     ```python
     BATCH_SIZE = 500_000
     batches = []
     current_batch = []
     current_count = 0
     for last_name, cnt in name_counts:
         if current_count + cnt > BATCH_SIZE and current_batch:
             batches.append(current_batch)
             current_batch = []
             current_count = 0
         current_batch.append(last_name)
         current_count += cnt
     if current_batch:
         batches.append(current_batch)
     ```
     Handle edge case: single last_name with >500K records (SMITH etc.) gets its own batch.

  4. For each batch:
     a. Read records from lake into LOCAL DuckDB table:
        ```sql
        CREATE OR REPLACE TABLE batch_data AS
        SELECT * FROM lake.federal.name_index
        WHERE last_name IN ('SMITH', 'JONES', ...)
        ```
        Use parameterized IN clause (join on a values table if list is too long).

     b. Load pre-trained model and create Linker:
        ```python
        linker = Linker("batch_data", settings="models/splink_model.json", db_api=DuckDBAPI(connection=conn))
        ```

     c. Run predict + cluster:
        ```python
        results = linker.inference.predict(threshold_match_probability=0.8)
        clusters = linker.clustering.cluster_pairwise_predictions_at_threshold(results, threshold_match_probability=0.85)
        ```

     d. Append results to a cumulative local table:
        ```python
        clusters_rel = clusters.as_duckdbpyrelation()
        if first_batch:
            conn.execute("CREATE TABLE all_results AS SELECT * FROM clusters_rel")
        else:
            conn.execute("INSERT INTO all_results SELECT * FROM clusters_rel")
        ```

     e. Log progress: batch N/total, records processed, clusters found, elapsed time.

     f. Clean up batch tables to free memory:
        ```python
        conn.execute("DROP TABLE IF EXISTS batch_data")
        ```

  5. After all batches complete, write results to DuckLake:
     ```sql
     CREATE OR REPLACE TABLE lake.federal.resolved_entities AS
     SELECT * FROM all_results
     ```

  6. Print final summary:
     - Total records processed
     - Total clusters
     - Multi-record matches (clusters with 2+ records)
     - Total elapsed time
     - Records/second throughput
