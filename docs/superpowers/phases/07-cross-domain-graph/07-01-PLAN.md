**Cross-domain bridge strategy:**
The unified graph needs edges that connect entities ACROSS domains:
1. **owner↔tx_entity**: Same person owns buildings AND appears in ACRIS transactions
2. **owner↔pol_entity**: Same person owns buildings AND donates/lobbies politically
3. **corp↔tx_entity**: Same corp in NYS filings AND in ACRIS transactions
4. **corp↔pol_entity**: Same corp officer AND political donor/vendor
5. **tx_entity↔pol_entity**: Same entity transacts property AND donates politically

The bridge key is: match entity names from each domain graph against resolved_entities, then connect entities that share a cluster_id.

**DuckPGQ constraint**: A property graph can have many vertex tables and many edge tables, but each edge table must reference exactly two vertex tables. Cross-domain edges must be defined as separate edge tables.

Build 3 bridge tables that connect domains via shared cluster_ids:

**Bridge 1: owner_is_tx_entity** — housing owners who are also ACRIS transaction entities
```sql
CREATE OR REPLACE TABLE main.graph_owner_tx_bridge AS
WITH owner_clusters AS (
    SELECT DISTINCT o.owner_id, r.cluster_id
    FROM main.graph_owners o
    JOIN lake.federal.resolved_entities r
        ON UPPER(TRIM(r.last_name || ' ' || COALESCE(r.first_name, ''))) = o.owner_id
           OR UPPER(TRIM(COALESCE(r.first_name, '') || ' ' || r.last_name)) = o.owner_id
    WHERE r.source_table IN ('hpd_registration_contacts', 'pluto')
),
tx_clusters AS (
    SELECT DISTINCT t.entity_name, r.cluster_id
    FROM main.graph_tx_entities t
    JOIN lake.federal.resolved_entities r
        ON UPPER(TRIM(r.last_name || ' ' || COALESCE(r.first_name, ''))) = t.entity_name
           OR UPPER(TRIM(COALESCE(r.first_name, '') || ' ' || r.last_name)) = t.entity_name
    WHERE r.source_table = 'acris_parties'
)
SELECT DISTINCT oc.owner_id, tc.entity_name AS tx_entity_name
FROM owner_clusters oc
JOIN tx_clusters tc ON oc.cluster_id = tc.cluster_id
```

**IMPORTANT**: This 56M-row resolved_entities join is VERY expensive. If OOM:
1. Stage resolved_entities subsets as temp tables first (WHERE source_table = 'X')
2. Or use simpler name matching: direct UPPER(TRIM()) equality between graph tables without resolved_entities (less accurate but much faster)

**Simpler fallback approach** (recommended if resolved_entities joins OOM):
```sql
-- Direct name matching between domain graphs (no cluster_id)
CREATE OR REPLACE TABLE main.graph_owner_tx_bridge AS
SELECT DISTINCT o.owner_id, t.entity_name AS tx_entity_name
FROM main.graph_owners o
JOIN main.graph_tx_entities t ON o.owner_id = t.entity_name

CREATE OR REPLACE TABLE main.graph_owner_pol_bridge AS
SELECT DISTINCT o.owner_id, p.entity_name AS pol_entity_name
FROM main.graph_owners o
JOIN main.graph_pol_entities p ON o.owner_id = p.entity_name

CREATE OR REPLACE TABLE main.graph_corp_pol_bridge AS
SELECT DISTINCT c.dos_id, p.entity_name AS pol_entity_name
FROM main.graph_corps c
JOIN main.graph_pol_entities p ON UPPER(TRIM(c.current_entity_name)) = p.entity_name
```

Direct name matching is exact (not fuzzy like Splink), but catches the most obvious cross-domain connections. It's also O(n*m) hash join instead of going through 56M resolved_entities rows.

**Try direct matching first** — it's fast and catches 80%+ of cross-domain links. If user wants fuzzy matching later, Phase 10 can add resolved_entities enhancement.

Also build a unified entity summary table for the entity_xray tool:
```sql
CREATE OR REPLACE TABLE main.graph_unified_entities AS
SELECT entity_name,
       MAX(CASE WHEN domain = 'housing' THEN 1 ELSE 0 END) AS in_housing,
       MAX(CASE WHEN domain = 'corporate' THEN 1 ELSE 0 END) AS in_corporate,
       MAX(CASE WHEN domain = 'transaction' THEN 1 ELSE 0 END) AS in_transaction,
       MAX(CASE WHEN domain = 'influence' THEN 1 ELSE 0 END) AS in_influence,
       COUNT(DISTINCT domain) AS domain_count
FROM (
    SELECT owner_id AS entity_name, 'housing' AS domain FROM main.graph_owners
    UNION ALL
    SELECT UPPER(TRIM(current_entity_name)), 'corporate' FROM main.graph_corps
    UNION ALL
    SELECT entity_name, 'transaction' FROM main.graph_tx_entities
    UNION ALL
    SELECT entity_name, 'influence' FROM main.graph_pol_entities
)
GROUP BY entity_name
HAVING COUNT(DISTINCT domain) >= 2
```

This gives multi-domain entities — the most investigatively interesting ones.

Add a unified property graph definition that includes vertex tables from all domains and cross-domain bridge edges:

```sql
CREATE OR REPLACE PROPERTY GRAPH nyc_unified
VERTEX TABLES (
    main.graph_owners PROPERTIES (owner_id, owner_name) LABEL Owner,
    main.graph_buildings PROPERTIES (bbl, housenumber, streetname, zip, boroid) LABEL Building,
    main.graph_tx_entities PROPERTIES (entity_name, tx_count, property_count, total_amount) LABEL TxEntity,
    main.graph_corps PROPERTIES (dos_id, current_entity_name, entity_type, county) LABEL Corp,
    main.graph_pol_entities PROPERTIES (entity_name, roles, total_amount, total_transactions) LABEL PoliticalEntity
)
EDGE TABLES (
    main.graph_owns
        SOURCE KEY (owner_id) REFERENCES main.graph_owners (owner_id)
        DESTINATION KEY (bbl) REFERENCES main.graph_buildings (bbl)
        LABEL Owns,
    main.graph_owner_tx_bridge
        SOURCE KEY (owner_id) REFERENCES main.graph_owners (owner_id)
        DESTINATION KEY (tx_entity_name) REFERENCES main.graph_tx_entities (entity_name)
        LABEL OwnerIsTransactor,
    main.graph_owner_pol_bridge
        SOURCE KEY (owner_id) REFERENCES main.graph_owners (owner_id)
        DESTINATION KEY (pol_entity_name) REFERENCES main.graph_pol_entities (entity_name)
        LABEL OwnerIsPolitical,
    main.graph_corp_pol_bridge
        SOURCE KEY (dos_id) REFERENCES main.graph_corps (dos_id)
        DESTINATION KEY (pol_entity_name) REFERENCES main.graph_pol_entities (entity_name)
        LABEL CorpIsPolitical
)
```

**DuckPGQ constraint check**: Each vertex table needs a unique PK column. graph_owners has owner_id, graph_tx_entities has entity_name, graph_corps has dos_id, graph_pol_entities has entity_name. Buildings has bbl. All good.
