     ```sql
     FROM lake.federal.resolved_entities
     WHERE cluster_id IN (...)
     GROUP BY source_table
     ORDER BY records DESC
     ```

  **Tool 2: top_crossrefs(min_tables, last_name_prefix)**

  ```python

  def top_crossrefs(
      min_tables: int = 5,
      last_name_prefix: str = "",
      limit: int = 50,
      ctx: Context = None,
  ) -> ToolResult:
      """Find people who appear in the most NYC data tables. Reveals the most
      cross-referenced individuals — useful for identifying high-profile persons,
      repeat offenders, major property owners, or politically connected individuals.
      Filter by minimum table count and optional last name prefix."""
  ```

  Implementation:
  1. Query resolved_entities for clusters appearing in many tables:
     ```sql
     SELECT cluster_id,
            MIN(last_name) as last_name,
            MIN(first_name) as first_name,
            COUNT(DISTINCT source_table) as table_count,
            COUNT(*) as total_records,
            ARRAY_AGG(DISTINCT source_table ORDER BY source_table) as tables
     FROM lake.federal.resolved_entities
     WHERE last_name IS NOT NULL
       AND (? = '' OR last_name LIKE ? || '%')
     GROUP BY cluster_id
     HAVING COUNT(DISTINCT source_table) >= ?
     ORDER BY table_count DESC, total_records DESC
     LIMIT ?
     ```

     ```
     |---|---|---|---|---|
     | SMITH | JOHN | 12 | 89 | acris_parties, campaign_contributions, ... |
     ...

     Total: 50 people shown (of X matching)
     ```
