  Create `src/dagster_pipeline/defs/resolved_entities_asset.py` with a Dagster @asset that wraps the batch processing logic:

  1. Define the asset with dependency on name_index:
     ```python
     from dagster import asset, AssetKey, MaterializeResult, MetadataValue, AssetIn

     @asset(
         key=AssetKey(["federal", "resolved_entities"]),
         group_name="federal",
         ins={"name_index": AssetIn(key=AssetKey(["federal", "name_index"]))},
         description="Probabilistic entity resolution across all lake tables using Splink",
     )
     def resolved_entities(context, name_index) -> MaterializeResult:
     ```

  2. Inside the asset function, replicate the core logic from `scripts/batch_resolve_entities.py`:
     - Connect to DuckLake via `_connect_ducklake()`
     - Get last_name frequency distribution from `lake.federal.name_index`
     - Pack into batches (500K per batch)
     - For each batch: load model, predict, cluster, append
     - Write to `lake.federal.resolved_entities`
     - Return MaterializeResult with metadata (total records, clusters, multi-record matches, duration)

  3. Import the helper functions rather than duplicating code. Refactor `batch_resolve_entities.py` to expose its core logic as importable functions:
     - `get_batch_plan(conn) -> list[list[str]]` — returns list of batches (each batch is list of last_names)
     - `process_batch(conn, batch_names, model_path) -> int` — process one batch, return cluster count
     - `write_to_lake(conn) -> dict` — write all_results to lake, return metrics

     Then the Dagster asset calls these functions and logs progress via `context.log`.

     ALTERNATIVELY, if refactoring the script is too complex, the asset can simply shell out:
     ```python
     import subprocess
     result = subprocess.run(
         ["python", "scripts/batch_resolve_entities.py"],
         capture_output=True, text=True, cwd="/app"
     )
     ```
     But prefer the importable approach for better Dagster integration (progress logging, metadata).

  4. Register in definitions.py:
     - Import resolved_entities from resolved_entities_asset
     - Add to all_assets list

  IMPORTANT: The asset takes ~51 min to run. That's fine — Dagster handles long-running assets.

  IMPORTANT: The model file `models/splink_model.json` must be accessible in the Docker container. It's in the project directory which should be mounted.
