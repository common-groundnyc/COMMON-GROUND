The existing self-join query is correct in structure but needs a safety cap for mega-landlords. An owner with 200 buildings creates 19,900 pairs — computationally expensive and inflates the graph with noise. Cap at owners with ≤100 buildings:

```sql
CREATE OR REPLACE TABLE main.graph_shared_owner AS
WITH owner_building_counts AS (
    SELECT owner_id, COUNT(*) AS cnt
    FROM main.graph_owns
    GROUP BY owner_id
    HAVING cnt >= 2 AND cnt <= 100
)
SELECT DISTINCT a.bbl AS bbl1, b.bbl AS bbl2
FROM main.graph_owns a
JOIN main.graph_owns b ON a.owner_id = b.owner_id AND a.bbl < b.bbl
WHERE a.owner_id IN (SELECT owner_id FROM owner_building_counts)
```

The cap of 100 buildings filters out city agencies (e.g., "NYC HOUSING AUTHORITY", "CITY OF NEW YORK") and holding companies that own hundreds of lots — these create quadratic edge explosion without adding investigative value. Owners with 2-100 buildings are the sweet spot for slumlord detection.
