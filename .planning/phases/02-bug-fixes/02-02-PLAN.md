**Bug 1 — property_history date sort crash** (line ~8225):
`sorted(deeds, key=lambda x: x["date"] or "")` — when `x["date"]` is `datetime.date` and fallback is `""`, Python can't compare the types. Appears 3 times: deeds sort, mortgages sort, satisfactions sort.
Fix: `key=lambda x: str(x["date"] or "0000-00-00")`

**Bug 2 — graph_corps duplicate vertex PKs** (line ~1439):
`graph_corps` has 417K rows but only 104K unique `dos_id` values (312K duplicates). The build query pulls from `nys_corporations` which has multiple filing records per corp. DuckPGQ's WCC algorithm requires unique vertex keys — duplicates crash `shell_detector`.
Fix: Add `GROUP BY dos_id` or `QUALIFY ROW_NUMBER() OVER (PARTITION BY dos_id ORDER BY initial_dos_filing_date DESC) = 1` to keep only the latest record per corp.

**Bug 3 — graph_has_violation orphaned edges** (line ~955):
13,701 BBLs in `graph_has_violation` don't exist in `graph_buildings`. These are buildings with HPD violations but not in HPD jurisdiction (demolished, merged lots, data entry errors).
Fix: Add `WHERE bbl IN (SELECT bbl FROM main.graph_buildings)` filter to the build query.

**Depends on**: 02-01 (infrastructure fixes) — duckpgq must load and S3 must work for graph rebuild to succeed.
