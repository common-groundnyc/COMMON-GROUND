**Prior work**: v1.0 name_index extracted person names from 44 tables using column pattern classification. The patterns discovered:
- SEPARATE: first_name + last_name columns
- COMBINED_COMMA: "LASTNAME, FIRSTNAME" in single column
- COMBINED_SPACE: "FIRSTNAME LASTNAME" in single column
- SINGLE_NAME: one name column (org or person)
- ACRIS_STYLE: party name with address fields
- CAMPAIGN_FINANCE: name + employer + occupation

**Entity types to classify**:
1. **person_name** ŌĆö any column(s) containing human names
2. **org_name** ŌĆö company, LLC, corp, business, restaurant, school names
3. **bbl** ŌĆö Borough-Block-Lot (10-digit NYC property identifier)
4. **address** ŌĆö street address, sometimes with city/state/zip
5. **license_number** ŌĆö DOB license, DCA license, taxi medallion, etc.
6. **permit_number** ŌĆö DOB job/permit numbers
7. **complaint_id** ŌĆö 311, HPD, DOB complaint identifiers
8. **registration_id** ŌĆö HPD registration, DCA registration
9. **bin** ŌĆö Building Identification Number (7 digits)
10. **zip** ŌĆö 5-digit ZIP code
11. **precinct** ŌĆö NYPD precinct number
12. **camis** ŌĆö restaurant CAMIS identifier
13. **dos_id** ŌĆö NYS Department of State corporation ID

<task type="auto">
  <name>Task 1: Introspect all tables and classify entity columns</name>
  <files>Output: entity column classification results (in-memory, passed to Task 2)</files>
  <action>
For each schema returned by `list_schemas`, call `list_tables` then `describe_table` for each table. Skip `_dlt_*` internal tables and `*_staging` schemas.

For each column in each table, classify it as an entity type based on column name patterns:

**person_name indicators**: name containing `first_name`, `last_name`, `firstname`, `lastname`, `name` (when paired with another name column), `respondent`, `owner_name`, `officer`, `applicant`, `petitioner`, `defendant`, `plaintiff`, `contact`, `agent`, `chairman`

**org_name indicators**: `corporationname`, `corporation_name`, `business_name`, `company`, `dba`, `entity_name`, `current_entity_name`, `employer`, `empname`, `vendor_name`, `lobbyist_name`, `client_name`

**bbl indicators**: column named `bbl`, or columns named `boro`/`borough` + `block` + `lot` that can be composed into BBL

**address indicators**: `address`, `addr`, `street`, `housenumber`, `streetname`, `location`, combined with `city`, `state`, `zip`

**license/permit**: `license`, `permit`, `job_number`, `job_`, `application_number`

**id columns**: `bin`, `camis`, `dos_id`, `registrationid`, `complaint_id`, `precinct`, `zipcode`, `zip`

Run queries against `information_schema.columns` to get all columns efficiently rather than calling describe_table 539 times:

```sql
SELECT table_schema, table_name, column_name, data_type
FROM information_schema.columns
WHERE table_catalog = 'lake'
  AND table_schema NOT LIKE '%staging%'
  AND table_name NOT LIKE '_dlt_%'
ORDER BY table_schema, table_name, ordinal_position
```

Then classify each column programmatically by name pattern matching.
